# Research Workflow

Цель — **100 источников (не превышать)**. Доводить до конца без участия пользователя.

## Оптимизация токенов

**Принцип:** основной контекст Opus — только принятие решений и финальная сборка. Вся "грязная работа" — в субагенты.

| Правило | Как |
|---|---|
| Субагенты на haiku | Все collector-агенты запускать с `model: "haiku"` — сбор URL, WebFetch, парсинг не требуют Opus |
| Верификация батчами | Проверка битых источников — один субагент haiku на весь батч, не по одному в основном контексте |
| Минимальный возврат | Субагенты возвращают **только список URL** (или текст для source_add), без рассуждений |
| Параллельные фоновые агенты | Все collector-агенты запускать с `run_in_background: true` одновременно |
| Точечное чтение | `notebook_get` — не читать полный ответ, только считать источники |
| Не повторять данные | Не дублировать в основном контексте то, что уже вернул субагент |
| `/compact` между этапами | После Шага 5 (добавление) вызвать `/compact` перед финальной проверкой |

## Шаг 0: Подготовка

**0а. Тип темы → YouTube-фильтр:**

| Тип | Примеры | Фильтр |
|---|---|---|
| Быстро меняющаяся | ИИ, крипто, LLM, маркетинг | `after:2023` |
| Умеренно динамичная | EdTech, fintech, e-commerce | `after:2022` |
| Медленно меняющаяся | история, биология, право | `after:2019` |
| Вечнозелёная | математика, физика, философия | без фильтра |

**0б. Веса каналов:**

| Канал | Эконом. | Наука | Гуманит. |
|---|---|---|---|
| Quick Search | ~5 | ~5 | ~5 |
| Official stats | 15 | 5 | 5 |
| Академические | 8 | 22 | 18 |
| Perplexity ×7 | ~45 | ~40 | ~40 |
| YouTube | 15 | 15 | 15 |
| Статистика/данные | 5 | 5 | 5 |
| Reddit+HN | 1 | 1 | 1 |
| Obsidian | 0-1 | 0-1 | 0-1 |

Добор до 100 через Perplexity или YouTube.

**0в. Проверить существующие источники:**
`notebook_get(notebook_id=...)` → извлечь URL/title в `existing_sources`. Бюджет = 100 − (уже есть).

**0г. Генерация углов Perplexity (адаптивные, не фиксированные):**

Вместо фиксированных 7 углов — генерировать их из темы. Алгоритм:
1. Определить 5-7 ключевых подтем/аспектов исследуемой темы
2. Для каждого аспекта сформулировать поисковый запрос
3. Обязательные слоты (если применимо к теме): статистика/данные, академический взгляд, тренды/прогнозы
4. Остальные слоты — специфичные для темы (не "РФ-данные" для квантовой физики, не "зарубежный опыт" если тема глобальная)

Примеры адаптации:
- "Российский EdTech" → рынок РФ, регулирование, конкуренты, технологии, метрики, пользовательский опыт, тренды
- "Квантовые вычисления" → физические основы, алгоритмы, hardware подходы, error correction, индустрия, сравнение с классическими, roadmap
- "Кофейный бизнес" → экономика, supply chain, обжарка/технологии, маркетинг, тренды потребления, регулирование, кейсы

**0д. Язык Perplexity-запросов:**
- РФ-специфичные аспекты → русский
- Глобальные аспекты → английский
- Глобальная тема (AI, finance) → все на английском, кроме РФ-специфичных

**0е. SCOPE — зафиксировать границы темы (ОБЯЗАТЕЛЬНО):**

Перед стартом сбора явно записать:
- **IN-SCOPE:** что точно входит в тему (ключевые параметры, пороги, контекст)
- **OUT-OF-SCOPE:** что точно НЕ входит (примеры того, что отсечь)

Примеры:
- Тема "Карьеры 1М+ руб/мес в России" →
  - IN: топ-менеджмент, партнёры консалтинга, инвестбанкинг, девелопмент, успешное предпринимательство, зарплаты >500к
  - OUT: профессии с доходом <500к, удалённая работа от 100к, US/UK карьеры, "выход из нищеты"
- Тема "Квантовые вычисления" →
  - IN: qubit технологии, алгоритмы, error correction, hardware, roadmap
  - OUT: квантовая механика без вычислений, популярная физика, sci-fi

Scope передаётся во ВСЕ субагенты и используется на Шаге 3 для фильтрации.

## Шаг 1: Запустить ВСЁ параллельно через subagents

Каждый канал сбора — **отдельный subagent** (Agent tool, `run_in_background: true`). Каждый возвращает только список URL. Это экономит токены основного контекста.

```
Основной контекст:
├── research_start(mode="quick", ...) — без subagent, просто запустить
│
├── Agent "perplexity-collector" (model: "haiku", run_in_background: true):
│   Задача: выполнить 7 Perplexity sonar-pro запросов (параллельно внутри агента),
│   собрать все citations, вернуть ТОЛЬКО дедупнутый список URL (без рассуждений).
│   Параметры запросов: model="sonar-pro", max_tokens=400, temperature=0,
│   return_citations=True, recency="year"
│   Углы: из Шага 0г (адаптивные, 5-7 штук, сгенерированные под тему)
│   Fallback: если угол дал < 3 citations → доп. запрос с другими ключевыми словами
│   Если бюджет позволяет — 8-й запрос: site:scholar.google.com PDF research paper
│   ⚠️ В промпт передать SCOPE из Шага 0е — агент должен отсеять off-scope citations
│
├── Agent "youtube-collector" (model: "haiku", run_in_background: true):
│   Задача: /youtube-search по 3-4 запросам (обзорный, экспертный, практический,
│   дискуссия). Фильтры: >10 мин, год из Шага 0, сортировка по views.
│   Вернуть ТОЛЬКО список YouTube URL.
│   ⚠️ В промпт передать SCOPE из Шага 0е + правила отсева:
│   - НЕ включать clickbait ("СЕКРЕТЫ", "за 10 минут", "ЛЕГКО", "%% обеднеют")
│   - НЕ включать видео с title явно за пределами scope
│   - Вернуть URL + title для каждого видео (для проверки в основном контексте)
│
├── Agent "academic-collector" (model: "haiku", run_in_background: true):
│   Задача: Semantic Scholar API + arXiv + OpenAlex запросы по теме.
│   OpenAlex: per_page=15, sort=cited_by_count:desc, is_oa:true,
│   from_publication_date по типу темы.
│   Вернуть ТОЛЬКО список URL академических статей.
│   ⚠️ arXiv/DOI — скачивать PDF через curl, добавлять через source_add(file).
│   НЕ добавлять arXiv как URL (NotebookLM не парсит). НЕ заменять на текст-абстракт.
│
├── Agent "reddit-hn-collector" (model: "haiku", run_in_background: true):
│   Задача: WebFetch HN Algolia API (hitsPerPage=20) + Reddit треды.
│   Собрать в единый markdown-текст (НЕ отдельные URL).
│   Вернуть ТОЛЬКО готовый текст для source_add(source_type="text").
│
├── Agent "stats-collector" (model: "haiku", run_in_background: true):
│   Задача: найти 5 источников с конкретными цифрами, данными, статистикой по теме.
│   Искать: отчёты (Statista, McKinsey, Deloitte, PwC, CB Insights),
│   дашборды (data.gov, Росстат, World Bank, OECD), инфографики с данными,
│   таблицы/графики с числовыми показателями.
│   Приоритет: первоисточники с raw data > аналитические отчёты > статьи с цифрами.
│   Perplexity sonar запросы: "[тема] statistics data 2024 2025", "[тема] market size numbers report".
│   Вернуть ТОЛЬКО список URL (до 5 штук) с пометкой какие данные содержит каждый.
│   ⚠️ В промпт передать SCOPE из Шага 0е — отсеять off-scope данные.
│
└── Официальная статистика — собрать прямые URL по теме (в основном контексте,
    т.к. требует знание темы)
```

## Шаг 2: Сбор результатов от subagents

Дождаться завершения всех agents → собрать URL в единый список.

## Шаг 3: Дедупликация, фильтрация и оценка релевантности

**3а. Дедупликация (3 этапа):**
1. Нормализация URL: убрать query-params (кроме `?id=`), `www.`, trailing `/`, lowercase домен
2. Удалить дубликаты по нормализованным URL
3. Дедупликация против `existing_sources` из Шага 0в

**3б. Жёсткая фильтрация по 7 правилам — субагент (model: "haiku"):**

Передать субагенту: SCOPE из Шага 0е + список URL с title/domain.
Субагент применяет 7 правил отсева:

1. **OFF-SCOPE** — title/domain за пределами scope → drop
2. **WRONG CONTEXT** — другая страна/рынок, если тема привязана к конкретному контексту → drop (иноязычные допустимы ТОЛЬКО если универсальный фреймворк/исследование)
3. **CLICKBAIT** — маркеры: "за 10 минут", "СЕКРЕТЫ", "ЛЕГКО", "%% обеднеют/разбогатеют", "никто не знает" → drop
4. **WIKIPEDIA-STUBS** — Wikipedia общих понятий без уникальных данных → drop (Wikipedia с конкретной статистикой по теме → keep)
5. **DUPLICATE CONTENT** — если уже есть 2+ источника с тем же типом контента (рейтинги, списки, обзоры) → drop лишние, оставить самые авторитетные (федеральные СМИ > отраслевые > образовательные лендинги > региональные перепечатки)
6. **РЕКЛАМНЫЕ ЛЕНДИНГИ** — продажа курсов/услуг без самостоятельной ценности → drop (маркеры: "поток", "запись на курс", "сопровождение")
7. **YouTube TITLE CHECK** — для каждого YouTube URL проверить title на соответствие scope, кликбейт-маркеры → drop при несоответствии

Субагент возвращает ТОЛЬКО два списка:
- **keep** — прошли все 7 правил
- **drop** — не прошли + номер правила (1-7) для каждого

Цель: **0% мусора** в ноутбуке. При сомнениях — drop (лучше не добрать, чем загрязнить).

**Blacklist (paywall — не загружаются):**
`wsj.com`, `ft.com`, `bloomberg.com`, `nytimes.com`, `economist.com`, `hbr.org`, `sciencedirect.com`, `springer.com/article`, `tandfonline.com`, `jstor.org`

Blacklist URL с ценным контентом → `WebFetch` → `source_add(source_type="text")`.

**Whitelist (пропускать верификацию битых, НО фильтрация по 7 правилам обязательна):**
`youtube.com`, `youtu.be`, `github.com`, `*.gov`, `*.gov.ru`, `*.edu`, `ncbi.nlm.nih.gov`, `medium.com`

⚠️ `arxiv.org` — НЕ добавлять как URL. Скачивать PDF через `curl`, добавлять через `source_add(source_type="file")`.
⚠️ `wikipedia.org` — НЕ в whitelist. Проверять по правилу 4 (WIKIPEDIA-STUBS).

Whitelist URL можно группировать в батч до 20, верификацию пропускать.

**Предварительный WebFetch НЕ делать.** Добавлять URL как есть → проверять битые после добавления (Шаг 5). Это экономит десятки WebFetch-вызовов. Битые починятся на этапе верификации.

## Шаг 4: Quick Search + бюджет

1. `research_status` → проверить завершение
2. **Таймаут:** 3 мин (3×60с). Не готов → продолжить, проверить в Шаге 6
3. `research_import` → источники добавлены
4. `notebook_get` → посчитать сколько добавил
5. Бюджет = 100 − (в ноутбуке) − 1 (Reddit+HN) − 0/1 (Obsidian)

## Шаг 5: Добавление по приоритету

**Порядок:** Official stats → Академические (как PDF file!) → Статистика/данные → Perplexity → YouTube → Reddit+HN (текст) → Obsidian (если есть и не пуст)

⚠️ **YouTube:** после source_add батча → notebook_get → проверить title каждого добавленного видео. Если title не соответствует scope → немедленно source_delete. YouTube — главный источник мусора.

**Адаптивный батч:** старт 10. После батча: 0 битых → +5 (макс 20) / 1-2 битых → без изменений / ≥3 битых → −5 (мин 5).

**Верификация после каждого батча (кроме whitelist):**
`notebook_get` → если title начинается с `http` → битый.

**Batch-починка битых — через субагент (model: "haiku"):**
Передать субагенту список битых source_id + URL → он делает:
1. `WebFetch` параллельно для всех битых
2. `source_delete` последовательно
3. `source_add(source_type="text")`
4. Возвращает ТОЛЬКО: сколько починено, сколько пропущено

**Retry:** ошибка source_add → 1 повтор через 5 сек. Неудача → пропустить.

**Счётчик:** после каждого батча `notebook_get` → приближаемся к 100 → стоп.

## Reddit + HN → текстовый источник

Не добавлять как URL. Subagent `reddit-hn-collector` возвращает готовый markdown:
```markdown
# Reddit + HN — обсуждения: ТЕМА
## Hacker News
### [название](url)
[содержание]
## Reddit
### [название](url)
[содержание]
```
→ `source_add(source_type="text", text=..., notebook_id=...)`

## Шаг 5.5: Сброс контекста

Вызвать `/compact` — сбросить накопленный контекст от батчей добавления. Дальше нужен только счётчик и список источников.

2. Quick Search отложен? → проверить `research_status`, импортировать
3. < 100 → добрать Perplexity/YouTube
4. > 100 → удалить наименее ценные (YouTube первыми)
5. Финальная дедупликация по title/URL
