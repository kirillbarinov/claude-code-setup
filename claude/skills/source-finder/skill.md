# source-finder

Поиск источников для NotebookLM через WebSearch (роль Perplexity) + NotebookLM Deep Research.

## Триггеры

Используй этот скилл когда пользователь:
- Просит найти источники по теме для ноутбука
- Говорит "найди источники", "поищи материалы", "добавь источники по теме"
- Использует `/source-finder`

## Алгоритм выполнения

### Шаг 1 — Уточни параметры (если не указаны)
- **Запрос:** что ищем?
- **Ноутбук:** в какой ноутбук добавляем? (покажи список через notebook_list если не указан)
- **Режим Deep Research:** fast (~30с, ~10 источников) или deep (~5мин, ~40 источников)?

### Шаг 2 — Perplexity (роль разведчика, 80-90% работы)
Используй Perplexity MCP для поиска источников. Запусти 2-3 запроса:
- Основной запрос по теме
- Запрос с фокусом на академические/профессиональные источники
- Запрос на свежие материалы 2025-2026

⚠️ ВАЖНО: Perplexity использовать ТОЛЬКО в обычном (fast/regular) режиме поиска. Deep Research через Perplexity ЗАПРЕЩЁН — для глубокого поиска используй только NotebookLM research_start на шаге 3.

Собери 5-15 лучших URL из результатов (предпочитай: официальные доки, лонгриды, исследования, GitHub).
Perplexity умеет обходить защиты сайтов и находить качественные первоисточники без галлюцинаций.

### Шаг 3 — NotebookLM Deep Research (10-20% работы)
Запусти встроенный поиск NotebookLM параллельно с WebSearch:
```
research_start(
  query="<тема>",
  notebook_id="<id>",
  source="web",
  mode="fast"  # или "deep" если пользователь выбрал
)
```
Затем дожидайся завершения:
```
research_status(notebook_id="<id>", task_id="<task_id>", max_wait=300)
```

### Шаг 4 — Покажи результаты пользователю
Выведи список найденных источников из обоих источников:
```
=== WebSearch нашёл: ===
1. [Название] — URL
2. ...

=== NotebookLM Deep Research нашёл: ===
1. Название — краткое описание
2. ...
```
Спроси: "Какие добавляем? (все / выбери номера / только WebSearch / только Deep Research)"

### Шаг 5 — Добавь выбранные источники

**Из WebSearch** — добавь через source_add:
```
source_add(
  notebook_id="<id>",
  source_type="url",
  urls=["url1", "url2", ...],
  wait=True
)
```

**Из Deep Research** — импортируй через research_import:
```
research_import(
  notebook_id="<id>",
  task_id="<task_id>",
  source_indices=[0, 1, 2, ...]  # индексы выбранных источников
)
```

### Шаг 6 — Отчёт
Сообщи сколько источников добавлено и из каких каналов.

## Важно
- WebSearch покрывает широкий интернет (как Perplexity) — используй для фундаментальных источников
- NotebookLM Deep Research точечный — используй для свежих деталей и специфических данных
- Всегда показывай результаты пользователю перед добавлением — пусть выберет
- Предпочитай качество количеству: лучше 5 отличных источников чем 20 средних
