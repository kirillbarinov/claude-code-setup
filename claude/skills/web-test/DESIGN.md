# /web-test — design spec

**Дата:** 2026-06-23
**Статус:** утверждён (вариант A — тонкий субагент)

## Цель

Глобальный скилл `/web-test`, который открывает текущую разработку в реальном
браузере, проходит golden path, наблюдает за console/network и выдаёт вердикт
PASS/FAIL. Главный приоритет — **минимум токенов в основном контексте**.

## Ключевое решение (изоляция MCP)

Вся работа с браузером выполняется в **тонком субагенте `web-test-runner`**,
а не в основной сессии. Это даёт строгую изоляцию стоимости:

- Схемы тулзов Chrome DevTools MCP грузятся **только** в контексте субагента,
  только в момент вызова скилла, и исчезают вместе с ним.
- Шумные дампы (accessibility-дерево, DOM, console, network) живут **только** в
  субагенте.
- В основную сессию возвращается **только** вердикт `PASS/FAIL + пути к
  артефактам`. Основной контекст не несёт ни схем, ни дампов — ни на старте,
  ни во время прогона, ни после.

`web-test-runner` — это НЕ `security-auditor`. Никакого полного QA-цикла,
edge-кейсов, регрессий, auto-fix loop, структурированного аудита. Только лёгкий
smoke. Паттерн скоупинга MCP на одного агента уже используется в
`security-auditor` (см. `~/.claude/agents/security-auditor.md`).

## Архитектура

```
/web-test
  → спавн субагента web-test-runner (tools whitelist: Bash, Read, Glob,
    mcp__chrome-devtools__*)
      1. DISCOVER → читает package.json / README / docker-compose:
                    как стартует фронт+бэк, какой порт
      2. SERVE    → dev-сервер уже поднят? нет → поднять в фоне,
                    ждать 200 OK на корне/health перед заходом браузером
      3. DRIVE    → выбор драйвера (auto-detect внутри субагента):
                      ├─ Chrome DevTools MCP (mcp__chrome-devtools__*) если есть
                      └─ иначе npx playwright CLI (fallback, дёшево)
      4. OBSERVE  → console errors + failed/slow network + pageerror + скриншот
                    финального состояния
      5. VERDICT  → вернуть в основную сессию: PASS/FAIL + пути к артефактам
  ← основной контекст: только вердикт. 0 схем MCP, 0 DOM-дампов.
```

## Драйвер браузера

Auto-detect внутри субагента, два пути (Playwright MCP сознательно исключён):

1. **Chrome DevTools MCP** (`mcp__chrome-devtools__*`) — если зарегистрирован в
   проекте. Интерактивный визуальный дебаг (DOM, network-панель, console, перф).
   Скилл его НЕ устанавливает сам.
2. **npx playwright CLI** (fallback, дефолт сегодня) — субагент генерит
   одноразовый `.spec.ts`, гоняет через Bash. В контекст субагента возвращается
   только вердикт + пути к trace/screenshot; сырой вывод остаётся на диске.

Важно: «Playwright CLI» (тест-раннер через npx) ≠ «Playwright MCP» (сервер).
Из auto-detect убран только MCP-сервер; CLI остаётся как fallback-драйвер.

## Dev-сервер

- Не запущен → субагент поднимает его сам в фоне.
- Уже крутится на известном порту → переиспользует, не трогает и не убивает.
- Перед заходом браузером — поллинг корня/health до 200 OK.

## Артефакты

```
<проект>/reports/qa/<дата-слаг>/
  ├─ screenshot.png   финальное состояние
  ├─ trace.zip        если CLI с трейсом
  └─ console.txt      ошибки/ворнинги/failed-requests
```

Скилл добавляет `reports/qa/` в `.gitignore` проекта (если ещё нет), чтобы не
мусорить в гит.

## Размещение

- Скилл: `~/.claude/skills/web-test/SKILL.md` (глобальный, во всех проектах).
- Субагент `web-test-runner`: `~/.claude/agents/web-test-runner.md`
  (глобальный), с `tools:` whitelist.

## Вход

`/web-test` без аргументов — авто-обнаружение. Опционально можно передать
подсказку (URL / какой флоу проверить) свободным текстом, скилл прокинет её в
промпт субагента.

## Out of scope (YAGNI)

- Полный QA-цикл / security-аудит / регрессии / auto-fix loop — это
  `security-auditor`, не дублируем.
- Установка/регистрация любых MCP-серверов — скилл их только использует, если
  уже есть.
- Playwright MCP как драйвер — исключён сознательно.
- Кросс-браузерные матрицы, CI-интеграция, видео-запись — позже, если понадобится.
